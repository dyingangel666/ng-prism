# A11y Plugin

`@ng-prism/plugin-a11y` runs axe-core accessibility audits on the rendered component canvas and displays violations, incomplete checks, and passes in a dedicated panel.

## What It Does

- Executes an axe-core audit on the rendered component after each variant switch or input change
- Displays violations grouped by impact level (critical, serious, moderate, minor)
- Shows incomplete checks (rules that need manual review)
- Lists passing rules
- Re-audits automatically with a 500 ms debounce whenever inputs change

## Install

```bash
npm install @ng-prism/plugin-a11y axe-core
```

## Configuration

```typescript
// prism.config.ts
import { defineConfig } from 'ng-prism';
import { a11yPlugin } from '@ng-prism/plugin-a11y';

export default defineConfig({
  plugins: [
    a11yPlugin({
      // optional: axe-core RunOptions applied globally
      runOnly: { type: 'tag', values: ['wcag2a', 'wcag2aa'] },
    }),
  ],
});
```

Options passed to `a11yPlugin()` are forwarded to axe-core as `RunOptions` for every audit. Omit them to run all rules.

## Per-Component Configuration

Override or extend audit rules for a specific component via `meta.a11y` on `@Showcase`:

```typescript
@Showcase({
  title: 'Icon Button',
  meta: {
    a11y: {
      rules: {
        'button-name': { enabled: true },
        'color-contrast': { enabled: true },
      },
      disable: ['landmark-one-main'],
    },
  },
})
```

Per-component `rules` are merged on top of the global plugin options. `disable` adds to any globally disabled rules.

## Violation Display

The panel shows violations sorted by impact severity:

| Impact | Color | Meaning |
|--------|-------|---------|
| `critical` | Red | Must fix — renders content inaccessible |
| `serious` | Orange | Should fix — significant barrier |
| `moderate` | Yellow | Consider fixing — affects some users |
| `minor` | Blue | Nice to fix — minor inconvenience |

Each violation card shows the rule ID, description, the affected DOM node, and a link to the axe-core rule documentation.

## Debounced Re-Audit

Audits are triggered after a 500 ms debounce following any input change. This prevents excessive audits during rapid slider or text input changes.

An audit is also triggered automatically when the active variant changes.

## Relationship to Built-in A11y Features

ng-prism core includes built-in accessibility tooling in a dedicated **A11y** panel:

- **Violations** sub-tab — score ring and violation list (powered by axe-core via this plugin)
- **Keyboard** sub-tab — keyboard navigation visualization overlay
- **ARIA Tree** sub-tab — live ARIA tree of the rendered component
- **Screen Reader** sub-tab — simulated screen reader output

The `@ng-prism/plugin-a11y` package provides the axe-core integration that powers the Violations tab. The other sub-tabs are part of ng-prism core.
